# 06 Site Structure Plan

## 1. 홈페이지 목적

박정은의 QA 경력과 프로젝트 경험을 한곳에서 보여주는 개인 대표 홈페이지를 제작한다.

QA 취업을 가장 중요한 목적으로 두고, 한화비전 SQA·SQE 경험을 중심에 배치한다. 유앤아이컴퓨넷의 장기근속과 운영지원 경험, 직접 진행한 웹사이트 기획·검수 경험도 함께 정리한다.

## 2. 최종 메뉴 구조

1. Home
2. About
3. Experience
4. Work
5. Contact

## 3. 섹션별 역할

### Home

방문자가 박정은의 주요 경력과 현재 지원 직무를 짧은 시간 안에 파악할 수 있도록 한다.

표시 내용:

- 메인 헤드라인
- 보조 문구
- 짧은 자기소개
- 경력 보기 버튼
- 프로젝트 보기 버튼

### About

QA 경력, 장기근속 경험, 웹사이트 제작 경험이 어떻게 연결되는지 설명한다.

단순한 경력 나열보다 실제 수행 업무와 일하는 방식을 보여주는 소개 영역으로 구성한다.

### Experience

회사별 근무기간과 역할, 주요 업무를 정리한다.

표시 경력:

- 유앤아이컴퓨넷
- 한화비전 SQA
- 한화비전 SQE

### Work

직접 정리하거나 제작·검수한 결과물을 프로젝트 카드 형태로 보여준다.

표시 프로젝트:

- QA 포트폴리오 사이트
- 쵸코네일 홈페이지
- 박정은 개인 대표 홈페이지

### Contact

채용 담당자나 방문자가 연락할 수 있는 정보를 제공한다.

이메일, QA 포트폴리오, GitHub 링크를 제공해 채용 문의와 작업 결과 확인이 가능하도록 구성한다.

## 5. 화면 구성 권장안

### 상단 영역

- 이름 또는 로고
- Home
- About
- Experience
- Work
- Contact

### 본문 순서

1. Home 히어로 영역
2. About 소개 영역
3. Experience 경력 타임라인 또는 카드

6. Footer

## 6. 다음 제작 단계

1. 확정한 Contact 정보와 링크를 홈페이지에 적용한다.
2. 홈페이지 기본 HTML 구조를 작성한다.
3. 각 Markdown 문서의 최종 문구를 HTML에 적용한다.
4. CSS로 레이아웃과 반응형 화면을 구성한다.
5. 브라우저에서 PC와 모바일 화면을 확인한다.
6. 문구, 링크, 버튼, 경력 기간을 최종 검수한다.
7. GitHub에 커밋하고 배포한다.
