# 08 Visual Direction Plan

## 1. 디자인 목적

QA 경력과 프로젝트 내용을 채용 담당자가 빠르게 확인할 수 있는 개인 대표 홈페이지를 만든다.

장식이나 화려한 효과보다 경력 기간, 실제 수행 업무, 프로젝트 담당 범위가 읽기 쉽게 보이는 것을 우선한다.

## 2. 전체 분위기

- 차분하고 정돈된 취업용 개인 홈페이지
- 신뢰감이 느껴지는 구성
- 과도하게 개발자 포트폴리오처럼 보이지 않는 디자인
- 긴 문장보다 제목, 요약, 핵심 업무가 먼저 보이는 구성
- 기존 QA 포트폴리오와 구분되지만 서로 연결감은 유지

## 3. 색상 방향

### 기본 배경

- 흰색 또는 매우 연한 회색
- 섹션 구분이 필요한 영역에는 연한 회색 배경 사용

### 주요 색상

- 짙은 네이비 또는 짙은 청회색
- 제목, 메뉴, 주요 버튼에 사용

### 강조 색상

- 차분한 파란색
- 링크, 기간, 버튼, 아이콘 등에 제한적으로 사용

### 피해야 할 색상

- 지나치게 밝은 원색
- 여러 강조색을 동시에 사용하는 구성
- 배경과 글자의 명도 차이가 적어 읽기 어려운 조합

## 4. 글꼴과 글자 구성

- 한글 가독성이 좋은 고딕 계열 글꼴 사용
- 본문보다 제목과 회사명을 명확하게 강조
- 본문 글씨를 너무 작게 만들지 않는다
- 한 문단이 지나치게 길어지지 않도록 폭을 제한한다
- 영문 메뉴와 한글 본문의 글꼴 분위기를 통일한다

## 5. 상단 메뉴

메뉴 순서:

1. Home
2. About
3. Experience
4. Work
5. Contact

구성 기준:

- 왼쪽에는 이름 또는 간단한 텍스트 로고 표시
- 오른쪽에는 메뉴 표시
- 현재 위치를 확인할 수 있는 강조 효과 적용
- 모바일에서는 메뉴가 화면 밖으로 넘어가지 않도록 구성

## 6. Home 화면

표시 순서:

1. 메인 헤드라인
2. 보조 문구
3. 짧은 자기소개
4. 주요 버튼

주요 버튼:

- 경력 보기
- 프로젝트 보기
- 연락하기

디자인 기준:

- 첫 화면에서 QA 경력을 바로 확인할 수 있어야 한다
- 지나치게 큰 사진이나 장식 이미지는 사용하지 않는다
- 텍스트 영역의 여백을 충분히 둔다

## 7. About 화면

- 최종 적용 소개 문구를 사용한다
- 한 문단을 너무 길게 표시하지 않는다
- QA 경험, 장기근속 경험, 웹 프로젝트 경험이 구분되어 보이도록 한다
- 단순한 자기소개보다 경력의 흐름이 보이도록 구성한다

## 8. Experience 화면

표시 경력:

1. 한화비전 SQE
2. 한화비전 SQA
3. 유앤아이컴퓨넷

디자인 기준:

- 최근 경력을 먼저 표시한다
- 회사명, 기간, 역할 요약, 주요 업무를 구분한다
- 타임라인 또는 세로형 카드 구조를 사용한다
- 주요 업무는 긴 문단보다 목록으로 표시한다
- 모바일에서는 카드가 한 줄에 하나씩 표시되도록 한다

## 9. Work 화면

표시 프로젝트:

1. QA 포트폴리오 사이트
2. 쵸코네일 홈페이지
3. 박정은 개인 대표 홈페이지

프로젝트 카드 구성:

- 프로젝트명
- 한 줄 설명
- 담당 범위
- 사용 도구
- 상세 보기 버튼

디자인 기준:

- 프로젝트 3개를 동일한 형식의 카드로 표시한다
- 개발 결과보다 기획, 검수, 수정 경험이 보이도록 한다
- 쵸코네일 홈페이지의 v1, v2 구분은 현재 문서 기준을 유지한다
- 개인 대표 홈페이지 버튼의 실제 연결 위치는 제작 단계에서 결정한다

## 10. Contact 화면

표시 내용:

- 이메일
- QA 포트폴리오
- GitHub 작업 기록

버튼:

- 이메일 보내기
- QA 포트폴리오 보기
- GitHub 작업 기록 보기

디자인 기준:

- 연락수단을 복잡하게 나열하지 않는다
- 이메일과 주요 링크를 버튼 또는 카드 형태로 명확하게 표시한다
- 개인정보는 확정된 공개 정보만 사용한다

## 11. 모바일 화면 기준

- 메뉴가 화면 밖으로 넘어가지 않게 한다
- 본문 좌우 여백을 확보한다
- Experience와 Work 카드는 한 줄에 하나씩 표시한다
- 버튼은 손가락으로 누르기 쉬운 크기로 만든다
- 긴 문장이 화면 밖으로 벗어나지 않게 한다
- 이메일과 URL이 잘리지 않도록 확인한다

## 12. 피해야 할 디자인

- 과도한 애니메이션
- 자동으로 계속 움직이는 배너
- 지나치게 작은 글자
- 긴 문장을 한 화면에 몰아넣는 구성
- 근거 없는 능력 점수나 그래프
- 실제 개발 경력으로 오해할 수 있는 표현
- 너무 많은 색상과 아이콘
- 내용을 가리는 장식 이미지

## 13. 최종 적용 방향

- 흰색과 연한 회색을 기본 배경으로 사용한다
- 짙은 네이비 또는 청회색을 주요 색상으로 사용한다
- Home은 QA 경력을 짧고 명확하게 보여준다
- Experience는 최근 경력부터 세로형으로 배치한다
- Work는 동일한 형식의 프로젝트 카드 3개로 구성한다
- Contact에는 이메일, QA 포트폴리오, GitHub 링크를 표시한다
- PC와 모바일 모두 가독성과 정보 전달을 우선한다
